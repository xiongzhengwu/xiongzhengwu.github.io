网络原理

HTTP与HTTPS的区别

HTTP：我们称为超文本传输协议，主要用来实现从网络传输超文本到本地浏览器，实现高效且准确的超文本传送。目前HTTP的版本为1.1
HTTPS：我们认为就是以安全为目标的HTTP通道，即在HTTP协议的基础上加入了SSL层，简称HTTPS。
SSL的主要作用分为以下两部分：
第一、为HTTP传输文件提供安全的传输通道
第二、确认网站的真实性，通过HTTPS访问的网站都需要获取到CA认证。

网络七层模型（了解）

从下到上的顺序
物理层
数据链路层
网络层
传输层（TCP、UDP）
会话层
表示层
应用层（HTTP FTP TFTP SMTP SNMP DNS TELNET HTTPS POP3 DHCP）

HTTP的请求机制

第一，由客户端（浏览器）发出请求
第二，通过网络传输，到指定的服务器地址
第三，服务器接收相关请求，并进行解析
第四，服务器将最终的结果响应回客户端
第五，客户端接收到服务器响应后，通过浏览器解析，最终将内容进行展现

客户端访问服务器，发出请求，有四部分构成

第一，请求方法（request method），默认为get方式。只有在表单提交和文件上传时为post方法。
第二，请求网址（request url），即在浏览器中输入的网路地址。
第三，请求头（request header），用来标识为一个浏览器正常的访问
第四，请求体（request body），用来传输到服务器端的相关数据

请求方法中get与post的区别

第一、使用get方式提交请求时，相关数据会作为url的参数被一起进行提交，我们可以直接在地址栏中观察到相关数据。而采用post方式请求的url不会包含这些数据，而是直接通过表单提交到服务器。
第二、get方式提交请求，对于提交的数据大小不能大于1024字节，而post方式提交数据，没有大小的限制。
总之，如果提交的信息比较敏感或者字节较大时，需要使用post方式；如果只是普通的请求使用get方式。

类变量与成员变量的区别

1、类变量是声明在类中，位于函数之外声明的变量。这些变量时公共的，即在类外使用类名.类变量的方式进行调用。我们一旦修改类变量的值，所有调用类变量的内容均发生改变。
2、成员变量，是声明在类中的构造函数中，通过self.变量来进行声明赋值的。这些成员变量是公有的，即在类外可以通过对象名.成员变量的方式进行调用。所有的对象都有自己的成员变量，也就意味着每个对象的成员变量都是独立的，不会相互干扰。
3、类变量是该类所有实例化对象所公有的，它会早于对象实例化之前被载入内容中。
成员变量都是跟随对象实例化的过程中，被声明出来的。它是伴随着对象的产生而产生，消亡而消亡的。
4、成员变量和方法，默认情况下都是公有的。也就是说在类外可以直接使用对象进行调用。我们可以通过为成员变量设置双下划线，将其标识为私有的成员变量，例如：self.__name = ‘tom’。私有的成员变量只能在类内进行直接使用，类外必须通过相关方法进行间接调用。

classmethod方法

在Python中，我们通常使用classmethod用来修饰一个方法，这个方法就被称为类方法，类方法通常用来操作类变量。类方法和类变量一样，都需要通过类名来调用。

staticmethod方法

在Python中，我们使用staticmethod来修饰函数，该函数被称为静态方法，首先，静态方法不需要使用对象参数或类参数作为默认参数。其次，静态方法通常只能调用类变量，而不可以使用成员变量。最后，在类外进行调用时，可以使用类名来调用。

实例方法、类方法与静态方法的区别
实例方法：第一个参数必须是实例对象（self）,通过它来传递实例的属性和方法，只能由实例对象来调用。
类方法：使用装饰器@classmethod，第一个参数必须是类对象（cls）,通过它来传递类的属性和方法。可以通过实例对象或类对象来调用，通常使用类对象调用。
静态方法：使用装饰器@staticmethod,没有默认参数（self和cls），但是方法体中不能使用类或实例对象的任何属性和方法。可以通过实例对象或类对象进行调用。

1、类方法和静态方法都可以通过对象实例或者类名来调用。
2、类方法主要针对的是类，可以通过继承和重新定义
3、静态方法不能被继承，无法被重写，可以被认为是全局函数。静态方法在使用时完全不会用到对象实例本身。静态方法逻辑上属于类，但是不会涉及到类中的属性和方法的操作。

方法重写

Python对XML文件的解析

1、XML解析的两种方式：DOM与SAX

DOM：文档对象模型，是Python处理XML文件的标准模式。一个DOM的解析器在解析XML文档时 ，一次性读入整个文档，把文档中的所有元素（标签和文本）保存在内存中的一个树形结构，然后我们可以利用DOM提供的各种函数，来读取或修改文档中的内容。
SAX：是一种时间驱动的API，利用SAX解析XML 。主要使用解析器负责读取XML文档，并向事件处理器发生事件，比如：元素开始、元素结束。常用在1）大型文档的处理，2）只需要文件的部分内容，或者从文件中得到特定的部分3）建立自己的对象模型。

dom与sax对比：
1、DOM解析需要读取整个文档到内存中，并且将所有元素保存到树形结构。那也就意味着，可以根据树形结构导航到任意节点，获取其相关内容。
2、SAX解析文档，将文档边读入，边解析的方式，不需要一次性读入全部文档内容，只需要读入一部分即可。同时，因为其边读入边解析的工作方式，能够适应大文件的解析，提高解析效率。

局部变量与全局变量

1、Python中，函数在调用的时候Python才会给函数一个内存地址，用来执行函数内部的相关代码。
2、Python程序在运行时，首先会将程序最外部的变量载入内容，并给其分配相关的内存空间。我们成这种变量为全局变量。
3、函数中声明的变量，只有在函数被执行时，才会被分配内存空间进行执行。我们成这种变量为局部变量。
4、Python程序执行的过程中，执行函数时首先会检查其函数内部的局部变量是否存在，如果存在就调用它，如果不存在就从外部全局变量中进行调用，如果仍然没有发现相关内容，则报错。
5、globals()与locals()
在Python中，通过globals()观察全局作用域中内容，包括：内置函数、自定义函数、全局变量；通过locals()观察局部作用域中的内容，包括：局部变量，嵌套函数（内部函数）inner
6、当函数中出现嵌套情况时，最内层函数执行调用变量，会采用“就近原则”即先查找自身范围内的变量，如果没有找到，则向上查找，直到程序最外部结束。
7、global与nonlocal
global用来修饰局部变量时，表示该变量将作为全局变量来使用。即不仅在函数中可以被调用，同时在文件最外部也可以被调用。
nonlocal用来修饰嵌套函数内部的变量，对父级作用域（外层函数）中的变量进行引用和修改，那么从该层(包括父级)以下都将调用该变量的值。注意，该关键字只能影响其直接父级函数。
使用global关键字修饰的变量之前可以并不存在
而使用nonlocal关键字修饰的变量在嵌套作用域中必须已经存在。

总结：在函数中定义的变量，我们成为局部变量。包括在函数内部定义的变量、作为参数传递进来的变量。局部变量起作用的范围，我们称为局部作用域。因为Python在调用局部变量时，需要给它分配内存地址，我们称为这个地址为局部命名空间。
在文件最外层定义的变量，我们成为全局变量，可以在文件中的任意位置进行调用。全局变量起作用的范围，我们称为全局作用域。Python在载入文件时，就需要给全局变量分配内存地址，这个地址我们称为全局命名空间。

Python运行时处理变量的步骤：
1、从Python解释器开始执行之后，就在内存中开辟里一个空间，每当遇到一个变量的时候，就把变量名和值之间对应的关系记录下来，但是当遇到函数定义的时候，解释器只是象征性的将函数名读入内存，表示知道这个函数存在了，至于函数内部的变量和逻辑，解释器根本不关心。
2、函数执行时，函数中的变量会储存在新开辟出来的内存中，函数中的变量只能在函数内部使用，并且会随着函数执行完毕，这块内存中的所有内容也会被清空

3、迭代器（Iterator）
1）概念：迭代器访问元素，是从第一个元素开始，依次访问后续元素直到结束。其中迭代器只能向前不能回退。其包含两个方法iter()和next()。

2、会将下一个元素进行返回

迭代器实现了工厂模式的对象，它在你每次访问下一个值的时候给你返回。

可迭代化（iterable）：表明相关的对象可以创建迭代器对象，同时他们自身都是可以进行for数据遍历的。我们可以使用isinstance()来判断对象是否为可迭代化

生成器
1、创建方式

2、使用
生成器我们通常可以认为一种特殊的迭代器，但是它不需要像迭代器一样去实现iter,next方法。只需要使用yield关键字，该关键字可以在每次调用生成器函数时起到暂停作用，并将其值进行返回。同时还会记录当前运行值的状态。

总结：迭代器和生成器
1、容器是一系列的元素集合，str、list、set、dict对象都可以看作是容器，容器都可以进行迭代（即用在while或for循环），凡是这样的容器我们都称为可迭代化对象（Iterable）。
2、可迭代化对象都实现了iter方法，该方法返回一个迭代器对象。
3、迭代器都拥有一个状态字段，用于记录下次迭代返回值。它实现了next和iter方法，迭代器不会一次性把所有的元素都加载到内存中，而是在需要的时候才生成返回结果。
4、生成器是一种特殊的迭代器，它的返回值不是通过return来返回的，而是通过yield来返回的。

copy()(浅拷贝)和deepcopy()（深拷贝）

1、浅拷贝：我们使用copy()函数将列表中的元素，进行复制，通常对于普通的列表（一维）能够实现完全的值传递，即将自身的副本赋值给另外一个列表。例如：

ls1 = [1,2,3,4,5]
ls2 = ls1.copy()  # 将ls1中的元素赋值给ls2,此时他们的值相等的
ls1[3] = 6        #此时，对ls1中索引位3的值进行修改，变为6

观察其结果，我们可以发现ls1 => [1,2,3,6,5],但是ls2 =>[1,2,3,4,5]没有发生改变
一旦，ls1的列表元素结构发生变化，出现子列表嵌套。

如：ls1 = [1,2,3,['one','two'],5]，这是再使用copy()对ls2赋值，我们进行如下修改：
ls1 = [1,2,3,['one','two'],5]
ls2 = ls1.copy()
ls1[3][0] = 'three'

我们观察其结果，可以发现ls1 => [1,2,3,[‘three’,’two’],5],同时ls2也会随之变化 ls2 => [1,2,3,[‘three’,’two’],5]。
总结：这种现象我们成为浅拷贝，copy()函数进行组合数据类型拷贝时，只能控制第一层元素进行值传递，一旦出现嵌套的子列表、子字典等内容，将不再拷贝其内容的值，而是直接将这个嵌套的内存地址，赋值给ls2。所以会造成一旦原列表改变子列表内容后，目标列表中对应的子列表也发生改变。
2、深拷贝：我们需要引入copy这个模块，调用deepcopy()函数。写法如下：

import copy
ls1 = [1,2,3,['one','two'],5]
ls2 = copy.deepcopy(ls1)
ls1[3][0] = 'three'

此时，我们发现ls1内容发生改变后，但是ls2并没有随之改变，那么也就是意味着deepcopy()将ls1中内容，包括嵌套里面的元素都进行了副本赋值，给ls2。因此，他们只是值相同，并没有指向同一个内存地址。

返回值：如果找到指定的字符串，则返回其索引下标位置的数字；如果找不到就会报错ValueError
区别：
find()与index()函数的区别
第一，两者在执行结果时类似的，如果找到指定的字符就会返回其索引下标位置的数字
第二，find()如果找不到指定的字符时，返回-1；而index()找不到指定的字符，raise ValueError.
第三，从效率上来讲，index的查询速度要优于find,因为在python中对于像字符串这样的有序数据类型，会产生一个索引对象来自动维护其序列，find()函数不管是否有这样的序列，都需要在字符串本身进行逐一查找，而index()会通过这个索引对象的映射方式，来快速找到对应的索引位。
